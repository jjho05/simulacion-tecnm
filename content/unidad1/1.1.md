# 1.1 Definición e importancia de la simulación en la Ingeniería

## Definición de Simulación

La **simulación** es una técnica numérica para conducir experimentos en una computadora, la cual involucra ciertos tipos de modelos matemáticos y lógicos que describen el comportamiento de un sistema (o algún componente del mismo) sobre períodos de tiempo.

### Definiciones Formales

**Según Shannon (1975):**
> "La simulación es el proceso de diseñar un modelo de un sistema real y llevar a cabo experiencias con él, con la finalidad de comprender el comportamiento del sistema o evaluar nuevas estrategias para el funcionamiento del sistema."

**Según Banks, Carson y Nelson (2010):**
> "La simulación es la imitación de la operación de un proceso o sistema del mundo real a través del tiempo."

### Componentes Clave

1. **Modelo:** Representación abstracta del sistema real
2. **Sistema:** Conjunto de entidades que interactúan para lograr un objetivo
3. **Experimento:** Prueba controlada del modelo bajo diferentes condiciones
4. **Análisis:** Interpretación de resultados para tomar decisiones

---

## Importancia en la Ingeniería

La simulación se ha convertido en una herramienta indispensable en la ingeniería moderna por las siguientes razones:

### 1. Reducción de Costos

**Ejemplo:** Una empresa automotriz puede simular el proceso de ensamblaje de un nuevo vehículo antes de construir la línea de producción física, ahorrando millones de dólares en inversión inicial.

**Beneficios cuantificables:**
- Reducción de 30-50% en costos de prototipado
- Ahorro de 20-40% en tiempo de desarrollo
- Disminución de 15-25% en errores de diseño

### 2. Experimentación Sin Riesgos

**Ejemplo:** Simulación de un reactor nuclear permite probar escenarios de emergencia sin poner en peligro vidas humanas o el medio ambiente.

**Aplicaciones críticas:**
- Entrenamiento de pilotos (simuladores de vuelo)
- Pruebas de seguridad en plantas químicas
- Evaluación de protocolos médicos de emergencia

### 3. Compresión del Tiempo

**Ejemplo:** Simular 10 años de operación de un sistema de manufactura en solo horas de cómputo.

**Ventajas:**
- Evaluación rápida de estrategias a largo plazo
- Identificación temprana de problemas futuros
- Planificación estratégica basada en proyecciones

### 4. Análisis de Sistemas Complejos

**Ejemplo:** Simulación del tráfico vehicular en una ciudad considerando miles de vehículos, semáforos, accidentes y condiciones climáticas simultáneamente.

**Sistemas típicamente simulados:**
- Redes de telecomunicaciones
- Cadenas de suministro globales
- Ecosistemas ambientales
- Mercados financieros

### 5. Optimización de Recursos

**Ejemplo:** Un hospital puede simular diferentes configuraciones de personal médico para minimizar tiempos de espera mientras optimiza costos laborales.

**Recursos optimizables:**
- Personal (cantidad y turnos)
- Equipamiento (número y ubicación)
- Inventarios (niveles y políticas de reorden)
- Capacidad instalada (expansiones y reducciones)

## Cuándo Usar Simulación

### Situaciones Ideales:

✅ **El sistema no existe aún** (está en fase de diseño)
✅ **Experimentar con el sistema real es costoso o peligroso**
✅ **El sistema es demasiado complejo para modelado analítico**
✅ **Se requiere visualización del comportamiento del sistema**
✅ **Se necesita evaluar múltiples escenarios alternativos**

### Situaciones Donde NO es Apropiada:

❌ El problema puede resolverse con sentido común
❌ El problema puede resolverse analíticamente (ecuaciones cerradas)
❌ Es más fácil experimentar directamente con el sistema
❌ No hay tiempo suficiente para desarrollar el modelo
❌ No hay datos disponibles para validar el modelo
❌ Los recursos computacionales son insuficientes

---

## Ejemplo Ilustrativo: Sistema de Atención Bancaria

### Problema:
Un banco desea determinar cuántas cajas de atención debe tener abiertas durante diferentes horas del día.

### Enfoque Tradicional (Sin Simulación):
- Observar y contar clientes
- Aplicar fórmulas de teoría de colas (si aplican)
- Implementar cambios y esperar resultados

**Limitaciones:**
- No considera variabilidad real
- Difícil evaluar múltiples escenarios
- Riesgo de implementar solución subóptima

### Enfoque con Simulación:
1. **Recolectar datos:** Tiempos entre llegadas y tiempos de servicio
2. **Construir modelo:** Representar el sistema en software
3. **Validar:** Comparar resultados del modelo con datos reales
4. **Experimentar:** Probar diferentes números de cajas (1, 2, 3, 4, 5...)
5. **Analizar:** Comparar tiempo de espera vs. costo de personal
6. **Decidir:** Seleccionar configuración óptima

**Resultados típicos:**
- Reducción de 40% en tiempo de espera promedio
- Ahorro de 15% en costos de personal
- Mejora de 25% en satisfacción del cliente

## Conclusión

La simulación es una herramienta poderosa que permite a los ingenieros:
- **Predecir** el comportamiento de sistemas antes de construirlos
- **Optimizar** operaciones existentes sin interrumpir la producción
- **Evaluar** múltiples alternativas de manera eficiente
- **Comunicar** ideas complejas mediante visualización

En la era de la transformación digital, la simulación se ha convertido en un componente esencial del proceso de toma de decisiones en ingeniería.

### Ejercicio 2: Análisis de Beneficios

**Escenario:** Una empresa manufacturera está considerando invertir en un estudio de simulación para optimizar su línea de producción.

**Datos:**
- Costo del estudio de simulación: $50,000
- Producción actual: 1,000 unidades/día
- Precio de venta: $100/unidad
- Mejora esperada en producción: 5%

**Preguntas:**
1. ¿Cuál es el incremento diario en ingresos si se logra la mejora?
2. ¿En cuántos días se recupera la inversión?
3. ¿Cuál es el beneficio anual si la planta opera 250 días/año?

**Solución:**

1. **Incremento diario:**
   - Producción adicional: 1,000 × 0.05 = 50 unidades/día
   - Ingresos adicionales: 50 × $100 = $5,000/día

2. **Recuperación de inversión:**
   - Días = $50,000 / $5,000 = 10 días

3. **Beneficio anual:**
   - Beneficio = $5,000 × 250 = $1,250,000/año
   - ROI = ($1,250,000 - $50,000) / $50,000 × 100% = 2,400%

## Tipos de Simulación en Ingeniería

### 1. Simulación de Eventos Discretos (DES)

**Definición:** Modela sistemas donde los cambios de estado ocurren en puntos discretos del tiempo.

**Aplicaciones típicas:**
- Manufactura
- Logística
- Servicios
- Call centers

**Ejemplo de código (Python/SimPy):**

```python
import simpy

def cliente(env, nombre, servidor):
    print(f'{nombre} llega en t={env.now}')
    with servidor.request() as req:
        yield req
        print(f'{nombre} inicia servicio en t={env.now}')
        yield env.timeout(5)  # Tiempo de servicio
        print(f'{nombre} termina servicio en t={env.now}')

env = simpy.Environment()
servidor = simpy.Resource(env, capacity=1)

# Crear clientes
env.process(cliente(env, 'Cliente 1', servidor))
env.process(cliente(env, 'Cliente 2', servidor))

env.run()
```

### 2. Simulación Continua

**Definición:** Modela sistemas donde las variables cambian continuamente en el tiempo.

**Aplicaciones típicas:**
- Sistemas de control
- Procesos químicos
- Dinámica de fluidos
- Sistemas eléctricos

**Ejemplo:** Simulación de un tanque de agua

```python
import numpy as np
import matplotlib.pyplot as plt

def tanque(nivel_inicial, flujo_entrada, flujo_salida, tiempo):
    niveles = [nivel_inicial]
    for t in range(1, tiempo):
        cambio = flujo_entrada - flujo_salida
        nuevo_nivel = niveles[-1] + cambio
        niveles.append(max(0, nuevo_nivel))  # No puede ser negativo
    return niveles

# Simular 100 minutos
niveles = tanque(nivel_inicial=50, flujo_entrada=5, flujo_salida=3, tiempo=100)
plt.plot(niveles)
plt.xlabel('Tiempo (min)')
plt.ylabel('Nivel (litros)')
plt.title('Simulación de Tanque de Agua')
plt.show()
```

### 3. Simulación de Monte Carlo

**Definición:** Usa muestreo aleatorio repetido para obtener resultados numéricos.

**Aplicaciones típicas:**
- Análisis de riesgo financiero
- Estimación de integrales
- Optimización estocástica

**Ejemplo:** Estimación de π

```python
import random

def estimar_pi(n):
    dentro_circulo = 0
    for _ in range(n):
        x = random.uniform(-1, 1)
        y = random.uniform(-1, 1)
        if x**2 + y**2 <= 1:
            dentro_circulo += 1
    return 4 * dentro_circulo / n

pi_estimado = estimar_pi(1000000)
print(f"π ≈ {pi_estimado}")  # Aproximadamente 3.14159
```

### 4. Simulación Basada en Agentes

**Definición:** Modela comportamiento individual de agentes autónomos y sus interacciones.

**Aplicaciones típicas:**
- Tráfico vehicular
- Propagación de epidemias
- Comportamiento de mercados
- Evacuaciones de emergencia

## Errores Comunes al Usar Simulación

### Error 1: No Definir Objetivos Claros

❌ **Incorrecto:** "Vamos a simular la fábrica"
✅ **Correcto:** "Determinar el número óptimo de máquinas para reducir tiempo de ciclo en 20%"

**Consecuencia:** Modelo demasiado complejo o que no responde las preguntas importantes.

### Error 2: Nivel de Detalle Inapropiado

❌ **Incorrecto:** Modelar cada tornillo en una línea de ensamblaje
✅ **Correcto:** Modelar estaciones de trabajo como unidades

**Regla:** El modelo debe ser tan simple como sea posible, pero tan complejo como sea necesario.

### Error 3: Ignorar la Validación

❌ **Incorrecto:** Confiar en resultados sin comparar con datos reales
✅ **Correcto:** Validar modelo con datos históricos antes de usarlo para decisiones

**Consecuencia:** Decisiones basadas en modelo incorrecto pueden ser costosas.

### Error 4: Usar Simulación Cuando No Es Necesaria

❌ **Incorrecto:** Simular un sistema simple con solución analítica conocida
✅ **Correcto:** Usar fórmulas de teoría de colas para sistemas M/M/1 simples

**Regla:** La simulación es la última opción, no la primera.

### Error 5: No Considerar Variabilidad

❌ **Incorrecto:** Usar tiempos promedio constantes
✅ **Correcto:** Usar distribuciones de probabilidad apropiadas

**Consecuencia:** Subestimación de tiempos de espera y cuellos de botella.

## Glosario de Términos Clave

| Término | Definición |
|---------|------------|
| **Modelo** | Representación abstracta de un sistema real |
| **Validación** | Proceso de asegurar que el modelo representa la realidad |
| **Verificación** | Proceso de asegurar que el código implementa correctamente el modelo |
| **Réplica** | Una ejecución del modelo con diferentes números aleatorios |
| **Warm-up period** | Tiempo inicial que se descarta para eliminar efectos transitorios |
| **Throughput** | Cantidad de entidades procesadas por unidad de tiempo |
| **Utilización** | Fracción del tiempo que un recurso está ocupado |
| **Bottleneck** | Recurso que limita la capacidad del sistema |
| **What-if analysis** | Evaluación de diferentes escenarios |
